# Padrões de UX do Vendeo v1

Status: Ativo
Versão: 1.0

Este documento define os padrões de experiência do usuário no Vendeo.

Ele garante consistência entre telas, simplicidade de uso e clareza nas interações.


--------------------------------------------------
1. PRINCÍPIO CENTRAL DE UX
--------------------------------------------------

O Vendeo deve ser extremamente simples de usar.

O usuário típico é um pequeno comerciante que possui pouco tempo e pouca paciência para interfaces complexas.

Toda decisão de UX deve priorizar:

simplicidade
clareza
rapidez de ação


--------------------------------------------------
2. REGRA DO FOCO NA AÇÃO
--------------------------------------------------

Cada tela deve ter uma ação principal clara.

Exemplos:

Nova campanha → criar campanha

Página da campanha → aprovar conteúdo

Lista de campanhas → gerenciar campanhas


Evitar múltiplas ações concorrentes.


--------------------------------------------------
3. REDUÇÃO DE CARGA COGNITIVA
--------------------------------------------------

O sistema deve evitar sobrecarregar o usuário com decisões.

Estratégias:

opções pré-definidas
campos guiados
textos curtos
interfaces limpas


O usuário não deve precisar pensar muito para usar o sistema.


--------------------------------------------------
4. USO DE ABAS
--------------------------------------------------

Conteúdos complexos devem ser separados em abas.

Exemplo na campanha:

Arte
Vídeo


Motivo:

o usuário raramente edita os dois conteúdos ao mesmo tempo.


--------------------------------------------------
5. INDICADORES DE PROGRESSO
--------------------------------------------------

O sistema deve sempre indicar o progresso da campanha.

Exemplo:

Arte ✓
Vídeo •


Ou:

1 de 2 conteúdos aprovados


Isso evita que o usuário esqueça de finalizar um conteúdo.


--------------------------------------------------
6. PADRÃO DE BOTÕES
--------------------------------------------------

Botões devem seguir padrões consistentes.

Exemplos:

Copiar arte
Baixar arte
Copiar conteúdo
Imprimir roteiro
Duplicar campanha


Evitar textos longos em botões.


--------------------------------------------------
7. FEEDBACK IMEDIATO
--------------------------------------------------

O sistema deve sempre informar ao usuário o resultado de suas ações.

Exemplos:

conteúdo copiado
arte gerada
vídeo gerado
campanha duplicada


Feedback rápido aumenta confiança no sistema.


--------------------------------------------------
8. EDIÇÃO CONTROLADA
--------------------------------------------------

A edição de conteúdo deve ser simples e focada.

Usuário pode:

editar texto
regenerar conteúdo
aprovar conteúdo


Evitar ferramentas complexas de edição.


--------------------------------------------------
9. PREVISIBILIDADE DA INTERFACE
--------------------------------------------------

Elementos devem sempre aparecer no mesmo lugar.

Exemplos:

ações da campanha sempre no topo

abas de conteúdo sempre na mesma posição

botões principais sempre visíveis


Isso reduz confusão.


--------------------------------------------------
10. EVOLUÇÃO DA UX
--------------------------------------------------

Novas funcionalidades devem respeitar os princípios de UX definidos neste documento.

A simplicidade do Vendeo deve ser preservada mesmo com evolução do produto.
