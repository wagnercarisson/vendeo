## 📋 CONTEXTO DA DECISÃO

### Problema: Edge Cases (Conveniência)
Durante planejamento de subsegmentação, identificamos que nem todos os lojistas se encaixam nas 4-5 subcategorias principais:

**Exemplo Real:**
- ❌ **Conveniência** vende bebidas, mas não é Adega/Distribuidora/Loja especializada
- ❌ **Posto de gasolina** vende bebidas, mas é multi-categoria
- ❌ **Bar** vende bebidas, mas foco é consumo no local

**Porcentagem Estimada:**
- 80% dos lojistas: Encaixam nas subcategorias principais
- 20% dos lojistas: Edge cases ("Outro")

---

## ✅ DECISÃO APROVADA

**Implementar schema com campo "Outro" + custom text field:**

### Schema Proposto

```sql
ALTER TABLE stores 
ADD COLUMN category TEXT,           -- Ex: 'bebidas_alcoolicas', 'mercearia'
ADD COLUMN subcategory TEXT,        -- Ex: 'adega', 'loja-bebidas', 'outro'
ADD COLUMN subcategory_custom TEXT; -- Campo livre quando subcategory = 'outro'
```

### Constraint de Validação

```sql
ALTER TABLE stores 
ADD CONSTRAINT check_subcategory_custom 
CHECK (
  (subcategory IS NULL OR subcategory != 'outro') OR 
  (subcategory = 'outro' AND subcategory_custom IS NOT NULL)
);
```

**Lógica:** Se escolher "Outro", é OBRIGATÓRIO preencher o campo custom.

### 2. Registry Loader (Responsável: @dev)

**Arquivo:** `lib/ai/prompts/registries/loader.ts`

**Lógica de Fallback:**
```typescript
export async function loadSegmentExpert(
  category: string, 
  subcategory?: string
): Promise<SegmentExpert> {
  
  // 1. Tenta variant específico (se não for "outro")
  if (subcategory && subcategory !== 'outro') {
    const variantPath = `${category}/variants/${subcategory}.yaml`;
    if (await exists(variantPath)) {
      return loadYaml(variantPath); // ✅ Conteúdo hiperpersonalizado
    }
  }
  
  // 2. Fallback para expert genérico (base)
  const basePath = `${category}/segment-expert.yaml`;
  return loadYaml(basePath); // ✅ Conteúdo contextualizado (70% qualidade)
}
```
