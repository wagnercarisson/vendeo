## 🔧 Implementação

### Lógica de Detecção (Backend)

```typescript
function detectSubcategoryKeywords(
  category: string,
  customValue: string
): { detected: boolean; suggestion?: string; message?: string } {
  
  const normalizedInput = customValue.toLowerCase().trim();
  const mappings = KEYWORD_MAPPING[category] || [];
  
  // Para cada grupo de keywords
  for (const { keywords, suggest } of mappings) {
    // Se QUALQUER keyword aparecer no input
    if (keywords.some(keyword => normalizedInput.includes(keyword))) {
      return {
        detected: true,
        suggestion: suggest,
        message: `Detectamos que seu negócio pode ser classificado como "${suggest}". 
                  Por favor, volte e escolha a opção correta para campanhas mais efetivas.`
      };
    }
  }
  
  return { detected: false };
}
```

### Resposta de Erro (API)

```json
{

  "message": "Detectamos que seu negócio pode ser classificado como \"emporio-cervejas\". Por favor, volte e escolha a opção correta para campanhas mais efetivas.",
  "suggestion": "emporio-cervejas"
}
```

## 🚧 Limitações Conhecidas

### 1. Falsos Positivos Potenciais

**Problema:** "Empório de Cachaças" contém "empório" (keyword).

**Solução:** Detecção smart:
- Se contém "cachaça" OU "whisky" OU "vodka" → aceita (especialização válida)
- Se contém APENAS "empório" → rejeita

**Implementação:**
```typescript
// Palavras que indicam especialização legítima (não devem rejeitar)
const LEGITIMATE_SPECIALIZATIONS = {
  emporio: ['cachaça', 'whisky', 'vodka', 'gin', 'destilados'],
  adega: ['destilados', 'licor'], // Se mencionar destilados, pode ser híbrido
};
```

### 2. Variações Ortográficas

**Problema:** "Emporio" (sem acento) vs "Empório" (com acento).

**Solução:** Normalização antes de detectar:
```typescript
const normalizedInput = customValue
  .toLowerCase()
  .normalize('NFD').replace(/[\u0300-\u036f]/g, '') // Remove acentos
  .trim();
```

### 3. Edge Cases Muito Específicos

**Problema:** "Empório de Cervejas e Vinhos" (híbrido).

**Decisão:** REJEITAR e sugerir escolher opção dominante:
- Se 70%+ cervejas → "Empório de Cervejas"
- Se 70%+ vinhos → "Adega"
- Se 50/50 → "Loja de Bebidas" (genérica)

**Razão:** Subsegmentação requer foco (não híbridos perfeitos).

## 🧭 Alinhamento Estratégico

**Por que isso importa:**

> "Subsegmentação perdida = perda de 30-37% de conversão. Se lojista escolhe 'outro' + 
> digita palavra-chave existente, sistema perde todo o valor da subsegmentação. 
> 
> Keyword detection garante que edge cases reais (conveniência, posto) usam 'outro', 
> mas casos que deveriam usar opções existentes (empório, adega) são guiados corretamente."

**ROI:** Evitar 15% de subsegmentação perdida = +2.1% conversão global (15% × 14% ganho subseg.)
