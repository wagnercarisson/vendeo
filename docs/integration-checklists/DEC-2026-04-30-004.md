## FASE 5: DECISÃO

### Opção Escolhida
**Deploy Phase 2.1 em Produção (Remote Supabase)**

### Justificativa
1. Migrations idempotentes (IF NOT EXISTS) garantem zero downtime
2. Sem breaking changes (colunas nullable, JSONB schemaless)
3. 6 constraints (incluindo 2 extras) garantem data quality desde o início
4. 3 partial indexes otimizam queries desde o deployment
5. Backward compatible (Phase 2.0 continua funcionando)
6. Rollback simples (9 DROP COLUMN atômico)

### Alternativas Rejeitadas
- **Esperar Docker local:** Delay desnecessário (migrations seguras)
- **Deploy sem constraints extras:** Dados ruins poderiam entrar (tokens negativos, weather inválido)
