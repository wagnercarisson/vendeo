## Backlog Positioning

Esta story foi reestabelecida em 2026-04-15 como a slice minima originalmente esperada na decomposicao do backlog. ## User Story

**Como** equipe de engenharia do Vendeo,
**Quero** adicionar ao schema de `stores` e `campaigns` os campos base do modelo V2,
**Para** permitir que as proximas stories de compatibilidade de campanha e marca avancem sem misturar essa entrega com weekly plan, approved assets ou runtime.

---

## Objective

Introduzir apenas as migrations base de `stores` e `campaigns` necessarias para a evolucao incremental do modelo V2. Esta story existe para isolar o fundamento de schema que continua faltando no backlog atual e que nao foi coberto pela 1.2.

---

## Scope IN

- Criar migration aditiva em `stores` para `brand_profile`, `brand_profile_version` e `brand_profile_updated_at`.
- Criar migration aditiva em `campaigns` para `domain_input`, `domain_input_version` e `legacy_content_type`.
- Preservar compatibilidade total com dados legados existentes, sem alterar runtime nesta entrega.

## Scope OUT

- Nao alterar `weekly_plan_items`.
- Nao criar `campaign_approved_assets`.
- Nao implementar normalizacao `info -> message`.
- Nao implementar fallback de `Brand Profile` na aplicacao.
- Nao alterar readers, writers, UI ou fluxo operacional atual.
- Nao executar backfill.
