## User Story

**Como** equipe de engenharia do Vendeo,
**Quero** implementar fluxo seguro `raw → validate → map → domain` com error handling robusto em todos os mappers de campanhas,
**Para** garantir que nenhum mapper quebra silenciosamente com dados inválidos, que erros em produção produzam mensagens úteis para debug, e que o novo `mapDomainToCampaignDb()` habilite operações de escrita tipadas.

---

## Objective

Refatorar `lib/domain/campaigns/mapper.ts` para substituir todos os usos de `.parse()` (que throw genérico) por `.safeParse()` com error handling explícito e mensagens úteis, e criar `mapDomainToCampaignDb()` como mapper de domínio → DB para operações de escrita.

---

## Scope IN

- Refatorar `mapDbCampaignToDomain()`:
  - Substituir `DbCampaignSchema.parse(data)` por `DbCampaignSchema.safeParse(data)`
  - try/catch com mensagem útil: `"[mapDbCampaignToDomain] Campo inválido: <field> — <issue>"`
  - Comportamento de throw mantido para compatibilidade (callers esperam throw)
- Refatorar `mapDbCampaignToAIContext()`:
  - Substituir `DbCampaignSchema.parse(data)` por `DbCampaignSchema.safeParse(data)`
  - Error handling com mensagem útil e throw
- Refatorar `mapAiArtToPreview()`:
  - Substituir `CampaignAISchema.parse(aiResponse)` por `.safeParse()`
  - Error handling com fallback (não throw — função é usada em UI client-side)
- Refatorar `mapAiReelsToPreview()`:
  - Substituir `CampaignReelsSchema.parse(aiResponse)` por `.safeParse()`
  - Error handling com fallback (não throw)
- Adicionar validação Zod ao input de `mapAiCampaignToDomain()`:
  - Validar `aiData` com `CampaignAISchema.safeParse()` antes de usar
  - Fallbacks documentados para campos ausentes (comportamento atual preservado)
- Criar `mapDomainToCampaignDb()`:
  - Input: `Campaign` (domain type)
  - Output: objeto pronto para `supabase.from('campaigns').update()`
  - Usa `buildCampaignContentTypeWrite()` para `content_type`/`legacy_content_type`
  - JSDoc com exemplo de uso
- Criar `lib/domain/campaigns/mapper.test.ts` com testes unitários para todas as funções refatoradas

## Scope OUT

- **Não** modificar `lib/domain/campaigns/service.ts` — continua chamando mappers com as mesmas assinaturas
- **Não** modificar `lib/campaigns/mapper.ts` (legacy layer) — já usa `.safeParse()` corretamente
- **Não** modificar `selectors.ts` (Story 2.5) ou API routes (Story 2.6)
- **Não** criar novos schemas Zod — usar apenas os existentes de `schemas.ts`
- **Não** alterar assinaturas das funções existentes — zero breaking changes para callers
- **Não** modificar componentes `.tsx` importadores — refactoring interno ao mapper

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| `DbCampaignSchema.safeParse()` rejeita campanhas existentes com campos extras/inválidos (ex: `content_type: "info"` em DB legado) | 🔴 ALTO — quebra listagens e página de campanha | `DbCampaignSchema` usa `CampaignReadableContentTypeSchema` que já aceita `"info"` — confirmar antes de refatorar. Testes com dados reais (incluir `content_type: "info"`) |
| Callers de `mapDbCampaignToDomain()` no service.ts não fazem try/catch — throw já é o comportamento esperado | 🟡 MÉDIO — se mudança para fallback silencioso, bugs ficam escondidos | Manter throw para funções de read (mapDbCampaignToDomain, mapDbCampaignToAIContext). Usar fallback apenas em funções de UI (mapAiArtToPreview, mapAiReelsToPreview) |
| `mapDomainToCampaignDb()` gera estrutura DB incorreta (campos faltando ou errados) | 🔴 ALTO — corrompe dados em produção | Validar output com `DbCampaignSchema.partial()` antes de retornar; incluir teste que verifica todos os campos chave |
| `mapAiArtToPreview()` e `mapAiReelsToPreview()` são usados em `CampaignPreviewClient.tsx` (UI) — fallback silencioso pode mostrar conteúdo em branco | 🟡 MÉDIO — degradação de UX sem erro visível | Fallback retorna campos vazios (string vazia, arrays vazios) — comportamento aceitável para UI; log no console para debug |
| Testes sintéticos passam mas dados reais do banco têm campos extras (ex: campos Supabase adicionados pelo sistema) | 🟡 MÉDIO — `DbCampaignSchema` usa `.strip()` implícito, campos extras são ignorados — não é risco real | Confirmar via `DbCampaignSchema.parse(realDbRow)` em teste |

### Pattern de error handling (referência)

**Para funções de read (throw mantido):**
```typescript
export function mapDbCampaignToDomain(data: unknown): Campaign {
  const result = DbCampaignSchema.safeParse(data);

  if (!result.success) {
    const firstIssue = result.error.issues[0];
    const path = firstIssue?.path?.join(".") ?? "unknown";
    const msg = firstIssue?.message ?? "validação falhou";
    throw new Error(`[mapDbCampaignToDomain] Campo inválido: ${path} — ${msg}`);
  }

  const raw = result.data;
  // ... mapping
}
```

**Para funções de UI (fallback, sem throw):**
```typescript
export function mapAiArtToPreview(campaign: Campaign, aiResponse: unknown): Partial<any> {
  const result = CampaignAISchema.safeParse(aiResponse);

  if (!result.success) {
    console.error("[mapAiArtToPreview] AI response inválida:", result.error.format());
    // Retorna fallback — não throw para não quebrar UI
    return {
      headline: campaign.product_name || "",
      body_text: "",
      cta: "",
      caption: "",
      hashtags: "",
      price_label: null,
    };
  }

  const ai = result.data;
  // ... mapping
}
```

**Para mapDomainToCampaignDb:**
```typescript
export function mapDomainToCampaignDb(campaign: Campaign): Record<string, unknown> {
  const contentTypeWrite = buildCampaignContentTypeWrite(campaign.content_type);
  return {
    product_name: campaign.product_name,

    price_label: campaign.price_label,
    audience: campaign.audience,
    objective: campaign.objective,
    product_positioning: campaign.product_positioning,

    campaign_type: campaign.campaign_type,

    headline: campaign.headline,
    body_text: campaign.body_text,
    cta: campaign.cta,
    ai_caption: campaign.ai_caption,
    ai_text: campaign.ai_text,
    ai_cta: campaign.ai_cta,
    ai_hashtags: campaign.ai_hashtags,
    ai_generated_at: campaign.ai_generated_at,
    // reels fields...
  };
}
```
