## 📋 Context

O Vendeo precisa transformar dados estruturados em decisões visuais criativas. Quando o Visual Reader (Motor 1) analisa uma imagem e retorna um `ImageProfile`, o **Intent Resolver (Motor 2)** recebe esses dados junto com informações da campanha e da identidade visual da loja, e decide:

- Qual tipo de composição usar (`directionType`)
- Qual tom emocional aplicar (`mood`)
- Como tratar o produto (`productTreatment`)
- Onde distribuir o texto (`textDistribution`)
- Quanto destacar o preço (`priceEmphasis`)
- Qual intensidade visual aplicar (`visualIntensity`)

**Source of Truth:** [4.2-BRIEF.md](./4.2-BRIEF.md)

---

## 🎯 Goal

Implementar o Intent Resolver com:
- **Entrada:** 25 campos estruturados (9 do ImageProfile + 7 da campanha + 9 da identidade visual)
- **Saída:** `CreativeDirection` com 6 campos de decisão criativa
- **Gate de decisão:** campanhas informativas não processam
- **System prompt otimizado:** via @prompt-eng com estrutura XML e chain-of-thought
- **Validações hard-coded:** post-LLM para garantir viabilidade
- **Modelo:** gpt-5.4 (full) para raciocínio criativo consistente
- **12 cenários de teste:** cobrindo matriz de backgroundType × objective

---

## 🏗️ Technical Approach

### Architecture Decision
- **New implementation:** `lib/ai/intent-resolver/` com 6 módulos separados
- **Model:** gpt-5.4 (full, não mini) — justificativa: raciocínio criativo requer precisão
- **Temperature:** 0.3 (criativo mas consistente)
- **Timeout:** 20s
- **Cache:** Adiado para Story 4.2.1 (opcional)

### AI Model Justification
- Motor 2 não processa imagem (só JSON estruturado)
- Raciocínio criativo requer modelo mais robusto
- Consistência nas decisões de `mood` e `directionType` é crítica
- Custo: ~$0.033/call vs ~$0.005 do mini (6.5x mais caro, mas vale a precisão)

### Validation Strategy
- **LLM gera direction inicial** baseado em contexto estruturado
- **Sistema aplica regras determinísticas** post-LLM para garantir viabilidade:
  - `DIRECTION_TYPE_MAP`: apenas tipos permitidos por `backgroundType`
  - `PRODUCT_TREATMENT_RULES`: apenas tratamentos viáveis por `backgroundType`
  - `targetOccupancy === "full"`: força composições que não requerem espaço ao redor

## 📥 Input Contract (25 fields)

### Group 1: `ImageProfile` (9 fields from Motor 1)

| Field | Type | Purpose |
|-------|------|---------|
| `backgroundType` | "transparent" \| "solid" \| "complex" \| "unknown" | Defines allowed `directionType` |
| `targetOccupancy` | "low" \| "medium" \| "high" \| "full" | `full` → prohibits decorative background |
| `sceneType` | "single_product" \| "multiple_products" \| "lifestyle_scene" \| "full_scene" \| "unclear" | Visual hierarchy (single vs multiple) |
| `visibility` | "clear" \| "partial" \| "obstructed" \| "unknown" | Obstructed product = avoid overlay |
| `framing` | "good" \| "tight" \| "distant" \| "unknown" | Distant = amplify prominence |
| `imageQuality` | "good" \| "acceptable" \| "poor" \| "unknown" | Poor = simple composition |
| `matchType` | "exact" \| "category_only" \| "none" | `none` = different strategy |
| `targetPosition` | "left" \| "center" \| "right" \| "top" \| "bottom" \| "mixed" \| "unknown" | Where product is → where text goes |
| `targetOrientation` | "vertical" \| "horizontal" \| "square" \| "mixed" \| "unknown" | Vertical vs horizontal layout |

**ImageProfile fields NOT used by Motor 2:**
- `targetBox` (used by Motor 4 — Renderer)
- `subjectCutoff`, `visualIsolation`, `safeExpansionPotential`, `focusClarity` (crop-related — out of scope)
- `reasoningSummary` (debug only)
- `detected`, `matchedTarget`, `confidence`, `relevantCount`, `ignoredElements` (additional context, not rules)

### Group 2: `CampaignData` (7 fields from database)

| Field | Type | Purpose |
|-------|------|---------|
| `content_type` | "product" \| "service" \| "message" | Gate: "message" doesn't process (returns empty direction) |
| `objective` | "promocao" \| "novidade" \| "queima" \| "sazonal" \| "reposicao" \| "combo" \| "engajamento" \| "visitas" \| "informativo" \| "institucional" \| "autoridade" | Derives context profile + influences mood |
| `price` | number \| null | `!== null` → `priceEmphasis` increases |
| `price_label` | string \| null | Custom label |
| `product_name` | string \| null | Context for LLM |
| `audience` | "geral" \| "jovens_festa" \| "familia" \| "premium_exigente" \| ... (12 values) | Influences mood |
| `product_positioning` | "popular" \| "medio" \| "premium" \| "jovem" \| "familia" | Influences mood and intensity |

### Group 3: `VisualSignature` (9 fields from database)

| Field | Source | Type | Purpose |
|-------|--------|------|---------|
| `primary_color` | visual_signatures | text | Base composition color |
| `secondary_color` | visual_signatures | text | Accent color |
| `logo_url` | visual_signatures | text \| null | Store logo |
| `store_name_typography` | visual_signatures | jsonb `{font, weight}` | Store name font |
| `signature_seed` | visual_signatures | text (UUID) | Seed for consistency across campaigns |
| `intensity_level` | visual_signature_profiles | "minimal" \| "balanced" \| "strong" | Base of `visualIntensity` |
| `composition_rules` | visual_signature_profiles | jsonb | Layout preferences |
| `typography_rules` | visual_signature_profiles | jsonb | Text rules |
| `color_rules` | visual_signature_profiles | jsonb | Color adaptation |

**Profile lookup:**
1. Derive `context_type` from `objective` (via `deriveContextProfile()`)
2. Fetch store's `visual_signature` (`store_id`)
3. Fetch matching `visual_signature_profile` (`signature_id` + `context_type`)
4. If no profile exists for this context → use "standard" as fallback

## 📤 Output Contract (6 fields)

### `CreativeDirection`

```typescript
type CreativeDirection = {
  directionType: "hero" | "frame" | "split-dynamic" | "overlay" | "stacked";
  mood: "clean" | "aggressive" | "playful" | "premium";
  productTreatment: "framed" | "background";
  textDistribution: "left" | "right" | "center" | "overlay";
  priceEmphasis: "low" | "medium" | "high";
  visualIntensity: "minimal" | "balanced" | "strong";
};
```

| Field | Values | How it's decided |
|-------|--------|-----------------|
| `directionType` | hero / frame / split-dynamic / overlay / stacked | LLM decides + post-LLM validation via DIRECTION_TYPE_MAP |
| `mood` | clean / aggressive / playful / premium | LLM based on `audience` + `product_positioning` + context profile |
| `productTreatment` | framed / background | LLM decides + post-LLM validation via PRODUCT_TREATMENT_RULES |
| `textDistribution` | left / right / center / overlay | LLM based on `targetPosition` + `directionType` |
| `priceEmphasis` | low / medium / high | `price !== null` increases; context profile defines magnitude |
| `visualIntensity` | minimal / balanced / strong | `intensity_level` from profile + adjustments by `imageQuality` |

### Field Details

#### 1. `directionType` — Composition type

| Value | When to use | Visual example |
|-------|------------|---------------|
| `hero` | Product alone in maximum prominence | Product centered, large, rest is text/price |
| `frame` | Product inside frame/container | Product framed, text around |
| `split-dynamic` | Product on one side, text on other | 50/50 or 60/40 layout |
| `overlay` | Text overlaid on image | Product occupies all, text on top |
| `stacked` | Product above, text below (or vice-versa) | Vertical stacked layout |

#### 2. `mood` — Emotional tone

| Value | Characteristics |
|-------|----------------|
| `clean` | Minimalist, white spaces, elegant |
| `aggressive` | Strong colors, high contrasts, urgency |
| `playful` | Fun, vibrant colors, informal |
| `premium` | Sophisticated, refined typography, sober |

#### 3. `productTreatment` — How to treat product image

| Value | Meaning |
|-------|---------|
| `framed` | Product stays inside delimited area/frame in composition |
| `background` | Entire image used as background, text/elements on top |

**Note:** `"cutout"` (product cutout) was removed — out of Epic 4 scope.

#### 4. `textDistribution` — Where text goes

| Value | When |
|-------|------|
| `left` | Product on right, text on left |
| `right` | Product on left, text on right |
| `center` | Text centered (product above/below or in background) |
| `overlay` | Text overlaid on image |

#### 5. `priceEmphasis` — How much to highlight price

| Value | When |
|-------|------|
| `low` | No price, or discreet price |
| `medium` | Price visible but not dominant |
| `high` | Price in maximum prominence (promotion) |

#### 6. `visualIntensity` — Overall composition intensity

| Value | Characteristics |
|-------|----------------|
| `minimal` | Few colors, lots of white space, clean |
| `balanced` | Balance between elements, standard |
| `strong` | Saturated colors, many elements, aggressive |

## 🔒 Hard-Coded Validation Rules (Post-LLM)

LLM generates initial `CreativeDirection`, but system applies deterministic rules to ensure viability.

### Table 1: DIRECTION_TYPE_MAP

Based on `backgroundType`, only certain `directionType` values are allowed:

| `backgroundType` | Allowed `directionType` | Logic |
|-----------------|------------------------|--------|
| `transparent` | hero, split-dynamic, stacked, frame | Clean PNG — maximum freedom |
| `solid` | frame, hero, split-dynamic | Flat background — direct composition |
| `complex` | overlay, frame, hero | Scene/environment — product needs prominence |
| `unknown` | frame, hero | Conservative fallback |

**Validation rule:**
```typescript
if (!DIRECTION_TYPE_MAP[profile.backgroundType].includes(direction.directionType)) {
  direction.directionType = DIRECTION_TYPE_MAP[profile.backgroundType][0]; // Use first from list
}
```

### Table 2: PRODUCT_TREATMENT_RULES

Based on `backgroundType`, only certain treatments are viable:

| `backgroundType` | Allowed `productTreatment` | Logic |
|-----------------|---------------------------|--------|
| `transparent` | framed, background | Both viable |
| `solid` | framed, background | Both viable |
| `complex` | background (only) | Complete scene — use entire image |
| `unknown` | framed (only) | Conservative fallback |

**Validation rule:**
```typescript
const rules = {
  transparent: ["framed", "background"],
  solid: ["framed", "background"],
  complex: ["background"],
  unknown: ["framed"],
};

if (!rules[profile.backgroundType].includes(direction.productTreatment)) {
  direction.productTreatment = rules[profile.backgroundType][0];
}
```

### Rule 3: targetOccupancy === "full"

If product occupies entire image, no space to add decorative background behind it.

```typescript
if (profile.targetOccupancy === "full" && direction.directionType === "hero") {
  // Force overlay or frame — hero implies space around
  direction.directionType = "overlay";
}
```

## 🧠 Context Profiles

### `deriveContextProfile()` Function

Derives `ContextProfile` from campaign `objective`:

```typescript
function deriveContextProfile(objective: CampaignObjective): ContextProfile {
  switch (objective) {
    case "sazonal":
      return "seasonal";
    case "promocao":
    case "combo":
      return "promotional";
    case "queima":
      return "urgency";
    case "institucional":
    case "autoridade":
      return "premium";
    default: // novidade, reposicao, engajamento, visitas, informativo
      return "standard";
  }
}
```

### Profile Characteristics

| Context Profile | When | Characteristics |
|----------------|------|----------------|
| `standard` | Common daily campaigns | Balanced, no extremes |
| `promotional` | Promotions and combos | Strong intensity, high price emphasis |
| `seasonal` | Holidays/special dates | Playful mood, balanced |
| `premium` | Institutional, authority | Clean/premium mood, minimal intensity |
| `urgency` | Stock clearance | Strong intensity, aggressive mood |
