**JSDoc Example:**
```typescript
/**
 * Typography specification for text elements in composition.
 * 
 * @property fontSize - Base font size in pixels (required)
 * @property fontWeight - Font weight (400, 600, 700, 900) — accepts number or string
 * @property fontFamily - Optional font family name (e.g., "Montserrat", "Sora")
 * @property color - Optional hex color (e.g., "#FF0000", "#2D3142")
 * @property lineHeight - Optional line height multiplier (e.g., 1.2, 1.4)
 * 
 * @example
 * {
 *   fontSize: 48,
 *   fontWeight: "700",
 *   fontFamily: "Montserrat",
 *   color: "#FF0000",
 *   lineHeight: 1.2
 * }

```

**Where to Apply:**
- `TypographySpecSchema` (line ~28-35) — Add fontFamily, color, lineHeight
- Remove `.strict()` from `CompositionSpecSchema` (line ~150)
- Remove `.strict()` from any nested schemas that reject extra fields

**Why `.coerce`?**
GPT sometimes returns `fontWeight: 700` (number) instead of `"700"` (string). `.coerce.string()` converts it automatically before validating against enum.

**Why remove `.strict()`?**
- Current: `.strict()` = reject ANY extra field (causes 100+ errors)
- After: Default mode = validate defined fields, ignore extras (0 errors)
- NOT `.passthrough()` = too loose, loses type safety

**Rationale:**
- GPT needs explicit instruction to avoid generating noise fields
- Encouraging useful fields (fontFamily, color, lineHeight) improves output consistency
- "do NOT include" is more effective than hoping GPT won't generate them

# Motor 3 (Visual Composer) — Schema Reference

## Schema Overview

Motor 3 uses Zod for runtime validation of GPT-5.4 outputs. Schema is designed to:
1. Accept useful fields GPT returns (fontFamily, color, lineHeight)
2. Coerce type mismatches (fontWeight number → string)
3. Ignore noise fields (promotion, renderingHints)

### Key Principles

- **Default mode** (not `.strict()`): Validates defined fields, ignores extras
- **Optional fields**: All new fields are optional (backward compatible)
- **Type safety**: Coercion preserves type safety while accepting variants

---

## Typography Spec Schema

```typescript
export const TypographySpecSchema = z.object({
  fontSize: z.number().int().positive(),
  fontWeight: z.coerce.string().pipe(z.enum(["400", "600", "700", "900"])),
  fontFamily: z.string().optional(),
  color: z.string().regex(/^#[0-9A-Fa-f]{6}$/).optional(),
  lineHeight: z.number().positive().optional(),
});
```

**Field Details:**

| Field | Type | Required | Validation | Purpose |
|-------|------|----------|------------|---------|
| `fontSize` | number | ✅ Yes | Positive integer | Base text size in pixels |
| `fontWeight` | string | ✅ Yes | Enum: "400", "600", "700", "900" | Font weight (coerced from number if needed) |
| `fontFamily` | string | ❌ No | Any string | Font family name (e.g., "Montserrat") |
| `color` | string | ❌ No | Hex format: /^#[0-9A-Fa-f]{6}$/ | Text color (e.g., "#FF0000") |
| `lineHeight` | number | ❌ No | Positive number | Line height multiplier (e.g., 1.2) |
