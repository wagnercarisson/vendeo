## Problem Statement

O **Motor 3 (Visual Composer)** está gerando **100+ erros de validação Zod** por execução:

```
[MOTOR-3][FALLBACK] Zod validation failed for output 1, using fallback structure
[MOTOR-3][ERROR] Zod validation error for output 1:
  - compositions.0.productName: Expected undefined but received string
  - compositions.0.layers.0.fontFamily: Expected undefined but received 'Montserrat'
  - compositions.0.layers.0.fontWeight: Expected '400' | '600' | '700' | '900' but received 700 (number)
  - compositions.0.elements.0.priceBadge: Unrecognized key
  - compositions.0.elements.0.lineHeight: Expected undefined but received number
```

### Impact Analysis

| Aspecto | Status | Observações |
|---------|--------|-------------|
| **Funcionalidade** | ✅ OK | Fallback gera 4 variações válidas |
| **Qualidade** | ⚠️ SUBOPTIMAL | Compositions podem ser genéricas |

**Severity:** Non-blocker mas impacta qualidade e manutenibilidade.

---

## Root Cause Hypothesis

**H1: Schema Drift (Most Likely)**  
GPT-5.4 evoluiu e agora retorna campos que não estavam no schema original:
- `productName` em `compositions[].productName`
- `priceBadge` em `compositions[].elements[]`
- Tipos numéricos onde schema espera strings (`fontWeight: 700` vs `'700'`)

**H2: Prompt Instruction Mismatch**  
O prompt do Motor 3 não alinha com o schema Zod explicitamente (não há JSON schema no prompt).

**H3: Schema Overly Strict**  
Campos úteis estão sendo rejeitados porque o schema foi projetado conservador demais.

---

## Proposed Solution Approach

### Option A: Update Schema to Match GPT Output (RECOMMENDED)
1. **Analisar** 10+ outputs do Motor 3 em logs de produção
2. **Identificar** campos consistentes que GPT retorna
3. **Atualizar** `lib/ai/visual-composer/contracts.ts` com campos novos
4. **Validar** que fallback não é mais necessário (0 errors)

**Pros:** Aceita evolução do modelo, melhora qualidade  
**Cons:** Pode precisar ajustar Motor 4 se novos campos afetarem rendering

### Option B: Constrain GPT with Schema in Prompt
1. **Extrair** JSON schema do Zod schema (`contracts.ts`)
2. **Incluir** schema no prompt do Motor 3 como instrução obrigatória
3. **Testar** se GPT respeita schema 100%

**Pros:** Força conformidade sem mudar código  
**Cons:** Pode limitar criatividade do GPT, requer prompt engineering

### Option C: Hybrid (Schema Update + Prompt Constraint)
1. Atualizar schema com campos úteis (productName, priceBadge)
2. Adicionar schema como instrução no prompt
3. Manter fallback como safety net

**Pros:** Melhor dos dois mundos  
**Cons:** Mais complexo, requer coordenação entre schema e prompt
