## 🎯 Objetivo

Implementar o **Motor 4 (Renderer)** — o engine de renderização programática que transforma `CompositionSpec` (coordenadas + layout) em artes finais PNG 1080×1350 prontas para download/aprovação.

**O que ele faz:**
- Recebe `CompositionSpec` (do Motor 3) + imagem do produto + dados da campanha + Visual Signature
- Renderiza arte final programaticamente usando Canvas API (node-canvas)
- Posiciona produto, texto, preço, badges, logo nas coordenadas exatas
- Aplica tipografia (5 famílias embedded para distinção entre lojas)
- Exporta PNG 1080×1350 (qualidade 90-95)
- Upload para Supabase Storage
- Retorna URL pública da arte

**O que ele NÃO faz:**
- Não gera layouts/coordenadas (isso é Motor 3)
- Não aplica cores da Visual Signature (isso é Motor 5 — backlog)
- Não renderiza vídeos (isso é Motor 7 — backlog)
- Não analisa imagens (isso é Motor 1)

## 📥 Contrato de Entrada

### Grupo 1: CompositionSpec (do Motor 3) — 40 campos

**Campos usados no Renderer:**

| Campo | Tipo | Para que serve no Renderer |
|-------|------|---------------------------|
| `layout.productArea` | `{x, y, width, height}` | Posicionar imagem do produto no canvas (crop/resize se necessário) |
| `layout.textArea` | `{x, y, width, height}` | Área para renderizar nome do produto |
| `layout.priceArea` | `{x, y, width, height}` | Área para renderizar preço formatado |
| `layout.badgeArea?` | `{x, y, width, height}` \| null | Área do badge de preço (se exists) |
| `typography.productName` | `{fontSize, fontWeight}` | Estilo do texto do nome do produto |
| `typography.price` | `{fontSize, fontWeight}` | Estilo do texto do preço |
| `typography.description` | `{fontSize, fontWeight}` | **NÃO usado** (description não é renderizado) |
| `decorative.priceBadge?` | `{shape, position, size, rotation}` | Renderizar badge decorativo (cloud, star, etc) |
| `decorative.storeIdentity` | `{type, position, size}` | Renderizar logo ou nome da loja |
| `decorative.promotionalTitle?` | `{text, position, fontSize, fontWeight}` | Título promocional (ex: "SUPER OFERTAS") |
| `spacing.padding`, `spacing.margins`, `spacing.gaps` | numbers | Ajustes finos de espaçamento (aplicados no Motor 3, respeitados aqui) |
| `hierarchy.primary/secondary/tertiary` | strings | Ordem de renderização (z-index) |

**Campos ignorados:** `spacing` (já aplicado nas coordenadas), `hierarchy` (ordem implícita: background → produto → badges → texto)

### Grupo 2: Product Image

```typescript
productImage: {
  url: string;        // URL pública ou signed URL (Supabase Storage)
  targetBox?: {       // Bounding box do produto (do Motor 1)
    x: number;        // normalizado 0-1
    y: number;
    width: number;
    height: number;
  } | null;
}
```

**Transformações necessárias:**

1. **Se `targetBox !== null`** (Motor 1 detectou produto):
   - Crop imagem para `targetBox` (coordenadas normalizadas → pixels)
   - Resize para fit em `layout.productArea` sem distorção (contain/cover)
   
2. **Se `targetBox === null`** (produto não detectado):
   - Usar imagem inteira
   - Centralizar em `layout.productArea`
   - Gerar alerta visual: "Produto não detectado — imagem completa usada"

**Libs:** Sharp para crop/resize (performance)

campaignData: {

## 🔒 Regras de Renderização

### Regra 1: Ordem de camadas (z-index)

```
1. Background (cor sólida #F5F5F5)
2. Product image (layout.productArea)
3. Decorative badges (decorative.priceBadge)
4. Text elements (layout.textArea, layout.priceArea)
5. Store identity (decorative.storeIdentity — logo ou nome)
6. Promotional title (decorative.promotionalTitle se exists)
```

### Regra 2: Text Overflow

Se texto exceder `layout.textArea`:
- Aplicar word wrap (quebra de linha)
- Se ainda exceder altura: truncar com "..." (ellipsis)
- Reduzir fontSize em 10% e tentar novamente (max 2 iterações)

### Regra 3: Badge Shapes

Renderizar 7 formas de badge suportadas:

| Shape | Descrição | SVG Path |
|-------|-----------|----------|
| `rounded-rect` | Retângulo com cantos arredondados (r=20px) | `roundRect()` |
| `cloud` | Nuvem irregular (explosão) | Custom path |
| `star` | Estrela 5 pontas | Custom path |
| `splash` | Splash/burst irregular | Custom path |
| `diamond` | Losango | Rotate 45° rect |
| `oval` | Elipse | `ellipse()` |
| `tag` | Tag com ponta | Custom path |

**Implementação:** Criar `lib/ai/renderer/shapes.ts` com funções para cada forma.

### Regra 4: Logo Positioning

Se `decorative.storeIdentity.type === "logo"`:
- Carregar imagem do `logo_url`
- Resize para `decorative.storeIdentity.size` (maintain aspect ratio)
- Posicionar em `decorative.storeIdentity.position`:
  - `top-left`: (40, 40)
  - `top-center`: ((1080 - width) / 2, 40)
  - `top-right`: (1080 - width - 40, 40)
  - `bottom-left`: (40, 1350 - height - 40)
  - `bottom-center`: ((1080 - width) / 2, 1350 - height - 40)
  - `bottom-right`: (1080 - width - 40, 1350 - height - 40)

Se `logo_url === null`:
- Renderizar `store_name` como texto (font size 20px, weight 700)

## 📐 Algoritmos Críticos

### Algoritmo 1: Crop & Resize Product Image

```typescript
async function cropAndResizeProduct(
  imageUrl: string,
  targetBox: {x, y, width, height} | null,
  productArea: {x, y, width, height}
): Promise<Buffer> {
  const img = await sharp(imageUrl);
  const metadata = await img.metadata();
  
  if (targetBox !== null) {
    // Crop para bounding box (coordenadas normalizadas → pixels)
    const cropX = Math.round(targetBox.x * metadata.width!);
    const cropY = Math.round(targetBox.y * metadata.height!);
    const cropWidth = Math.round(targetBox.width * metadata.width!);
    const cropHeight = Math.round(targetBox.height * metadata.height!);
    
    img.extract({ left: cropX, top: cropY, width: cropWidth, height: cropHeight });
  }
  
  // Resize para fit em productArea (contain mode - sem distorção)
  img.resize({
    width: productArea.width,
    height: productArea.height,
    fit: 'contain',
    background: { r: 0, g: 0, b: 0, alpha: 0 }, // transparent
  });
  
  return img.toBuffer();
}
```

### Algoritmo 2: Render Badge Shape

```typescript
function renderBadge(
  ctx: CanvasRenderingContext2D,
  badge: {shape, position, size, rotation}
) {
  ctx.save();
  
  // Aplicar rotação se definida
  if (badge.rotation) {
    ctx.translate(badge.position.x + badge.size.width / 2, badge.position.y + badge.size.height / 2);
    ctx.rotate((badge.rotation * Math.PI) / 180);
    ctx.translate(-(badge.size.width / 2), -(badge.size.height / 2));
  } else {
    ctx.translate(badge.position.x, badge.position.y);
  }
  
  // Renderizar forma específica
  switch (badge.shape) {
    case 'cloud':
      renderCloudShape(ctx, badge.size);
      break;
    case 'star':
      renderStarShape(ctx, badge.size);
      break;
    // ... outras formas
  }
  
  ctx.fillStyle = '#3B82F6'; // cor neutra temporária
  ctx.fill();
  ctx.restore();
}
```

### Algoritmo 3: Word Wrap com Ellipsis

```typescript
function renderTextWithWrap(
  ctx: CanvasRenderingContext2D,
  text: string,
  area: {x, y, width, height},
  fontSize: number
): void {
  ctx.font = `${fontSize}px Inter`;
  const words = text.split(' ');
  const lines: string[] = [];
  let currentLine = '';
  
  // Word wrap
  for (const word of words) {
    const testLine = currentLine + (currentLine ? ' ' : '') + word;
    const metrics = ctx.measureText(testLine);
    
    if (metrics.width > area.width) {
      lines.push(currentLine);
      currentLine = word;
    } else {
      currentLine = testLine;
    }
  }
  lines.push(currentLine);
  
  // Truncar se exceder altura
  const lineHeight = fontSize * 1.2;
  const maxLines = Math.floor(area.height / lineHeight);
  
  if (lines.length > maxLines) {
    lines.splice(maxLines);
    lines[maxLines - 1] += '...';
  }
  
  // Renderizar linhas
  lines.forEach((line, i) => {
    ctx.fillText(line, area.x, area.y + (i + 1) * lineHeight);
  });
}
```

## 🚀 Performance Requirements

| Métrica | Target | Método |
|---------|--------|--------|
| Tempo por variação | <3s | Canvas API + Sharp optimizations |
| Tempo total (4 variações) | <8s | Worker pool (parallel) |
| Memória por render | <150MB | Garbage collection explícito + buffer cleanup |
| Tamanho PNG | 600-900KB | Qualidade 90-95 |
| Taxa de sucesso | >99% | Fallbacks em caso de erro (font, image load) |

**Estratégia de paralelização:**
- Worker pool com 4 workers (1 por variação)
- Cada worker renderiza 1 variação independentemente
- Main thread aguarda Promise.all()
