## User Story

**Como** equipe de engenharia do Vendeo,
**Quero** consolidar tipos de domínio esparsos em `lib/domain/campaigns/types.ts` como fonte única de verdade inferida de schemas Zod,
**Para** eliminar duplicações, prevenir inconsistências de tipo e garantir que todos os tipos de domínio derivam dos contratos de validação estabelecidos na Story 2.1.

---

## Objective

Refatorar `lib/domain/campaigns/types.ts` para que `Campaign`, `ContentType` e tipos relacionados sejam inferidos de schemas Zod (Story 2.1), exportar um `ContentType` canônico fechado, e marcar como `@deprecated` os tipos duplicados encontrados em `lib/campaigns/types.ts` e `lib/campaigns/contracts.ts`.

---

## Scope IN

- Refatorar `lib/domain/campaigns/types.ts`:
  - `Campaign` → re-export/alias de `CampaignDomain` (`z.infer<typeof CampaignDomainSchema>`)
  - `ContentType` → alias canônico de `CampaignCanonicalContentType` (`"product" | "service" | "message"`)
  - `Objective` → alias de `CampaignObjective` de `lib/constants/strategy`
  - Manter `CampaignCanonicalContentType`, `CampaignReadableContentType`, `CampaignListItem`, `CampaignDetail`, tipos UI (`ContentState`, `ActiveTab`, `ViewMode`) inalterados
- Marcar `@deprecated` (JSDoc) os tipos em `lib/campaigns/types.ts` (`Campaign` camelCase)
- Marcar `@deprecated` (JSDoc) os tipos em `lib/campaigns/contracts.ts` (`CampaignStatus`, `CampaignStrategy`)
- Verificar e confirmar que todos os importadores existentes continuam funcionando (typecheck)

## Scope OUT

- **Não** criar schemas Zod para `CampaignListItem`, `CampaignDetail`, `CampaignContext` — sem schema disponível
- **Não** migrar ou remover o arquivo `lib/campaigns/types.ts` ou `lib/campaigns/contracts.ts`
- **Não** modificar `lib/constants/strategy.ts` (fonte canônica de `CampaignObjective`)
- **Não** criar tipo `Strategy` — nenhum schema Zod existe; `CampaignStrategy` em contracts.ts é legacy uppercase; definição canônica é fora do escopo desta story
- **Não** atualizar componentes React (`.tsx`) — apenas arquivos `.ts` de domínio
- **Não** alterar a lógica de `mapper.ts` ou `selectors.ts`

- `CampaignDetail extends Campaign {}` — empty extension
- `CampaignContext`, `CampaignAIOutput` — interfaces manuais
- UI types: `ContentState`, `ActiveTab`, `ViewMode`

**`lib/domain/campaigns/schemas.ts`** (Story 2.1) — Exporta:
- `CampaignDomainSchema` — schema Zod completo
- `export type CampaignDomain = z.infer<typeof CampaignDomainSchema>` — JÁ EXISTE
- `CampaignCanonicalContentTypeSchema = z.enum(["product", "service", "message"])`
- `CampaignReadableContentTypeSchema = z.enum(["product", "service", "info", "message"])`

**Ação principal:** Em `types.ts`, substituir `export interface Campaign { ... }` manual por:
```typescript
export type { CampaignDomain as Campaign } from "./schemas";
// ou equivalentemente:
import type { CampaignDomain } from "./schemas";
export type Campaign = CampaignDomain;
```

### Novos exports a adicionar em `lib/domain/campaigns/types.ts`

// Inferido de schema — ContentType canônico (fechado)

### Arquivos duplicados a marcar @deprecated

**`lib/campaigns/types.ts`** — `Campaign` interface com campos camelCase (storeId, productName...) — LEGACY, não usado em domain layer novo.
```typescript

```

**`lib/campaigns/contracts.ts`** — `CampaignStatus`, `CampaignStrategy`, `CampaignObjective`:
