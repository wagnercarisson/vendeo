## User Story

**Como** equipe de engenharia do Vendeo,
**Quero** adicionar ao schema de `weekly_plan_items` os campos de dominio planejado,
**Para** separar planejamento de execucao sem mexer ainda no comportamento do weekly plan.

---

## Objective

Introduzir apenas o schema necessario para que o weekly plan carregue a intencao de dominio futura sem alterar o fluxo atual de planejamento. Esta story existe para isolar o risco semantico de `weekly_plan_items` em um artefato pequeno e validavel.

---

## Scope IN

- Criar migration para `weekly_plan_items.target_content_type text`.
- Criar migration para `weekly_plan_items.target_domain_input jsonb not null default '{}'::jsonb`.
- Garantir compatibilidade total com o uso atual de `weekly_plan_items.content_type` como formato operacional legado.

## Scope OUT

- Nao alterar promocao de item de plano para campanha.
- Nao fazer backfill de `target_content_type`.
- Nao implementar regras de aplicacao para preencher `target_domain_input`.
- Nao alterar UI, prompts ou renderer.

---

## Acceptance Criteria

```gherkin
GIVEN o banco atual do Vendeo
WHEN a migration desta story for aplicada
THEN `weekly_plan_items` passa a conter `target_content_type` e `target_domain_input`
AND `weekly_plan_items.content_type` permanece intacto como campo legado de formato
```
