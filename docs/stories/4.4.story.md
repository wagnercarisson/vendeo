## 📋 Context

O Vendeo gera artes para campanhas de lojas físicas. Após análise da imagem (Motor 1), definição da direção criativa (Motor 2), e composição de layouts (Motor 3), o sistema precisa **renderizar as artes finais** prontas para download.

O **Renderer v2** é o **Motor 4** da pipeline de composição visual. Ele recebe `CompositionSpec` com coordenadas precisas e responde: "aqui está a arte PNG 1080×1350 pronta!" — renderizando programaticamente todas as camadas (produto, texto, preço, badges, logo) e fazendo upload para Supabase Storage.

**Source of Truth:** [4.4-BRIEF.md](./4.4-BRIEF.md)

**Critical Dependencies:**
- Story 4.1 (Visual Reader) MUST be complete — precisa do `targetBox`
- Story 4.2 (Intent Resolver) MUST be complete — contexto criativo
- Story 4.3 (Visual Composer) MUST be complete — precisa do `CompositionSpec`

## 🎯 Goal

Implementar o **Motor 4 (Renderer)** com:
- Contrato de entrada: `CompositionSpec` (40 campos) + Product Image + Campaign Data + Visual Signature
- Renderização programática: Canvas API (node-canvas) server-side
- Processamento de imagem: Sharp para crop/resize
- Font library: 5 famílias embedded (Inter default MVP)
- Badge shapes: 7 formas suportadas (cloud, star, splash, etc)
- Output: PNG 1080×1350, qualidade 90-95, 600-900KB
- Upload: Supabase Storage (`campaigns/{id}/variation-{i}.png`)
- Performance: 4 variações em <8s (parallel worker pool)
- 8 fixtures de teste cobrindo cenários críticos

## 🏗️ Technical Approach

### Architecture Decision
- **New implementation:** `lib/ai/renderer/` com rendering engine completo
- **No legacy code:** Esta é a primeira implementação do Motor 4
- **Stack:** node-canvas + Sharp + Supabase Storage
- **Deterministic:** Sem AI — rendering baseado em coordenadas exatas do Motor 3

### Canvas Rendering
- **Canvas API** (node-canvas) para tipografia precisa e rendering server-side
- **Canvas dimensions:** 1080×1350 pixels (Instagram/Facebook portrait)
- **Output format:** PNG (quality 90-95)
- **Backlog:** WebP optimization (Story 4.9)

### Image Processing
- **Sharp** para crop/resize otimizado
- Crop para `targetBox` (se não null)
- Resize com fit 'contain' (sem distorção)
- Background transparente

### Font Management
- **5 famílias embedded:** Inter, Poppins, Bebas Neue, Montserrat, Roboto Condensed
- **MVP:** Inter como default para todas as artes
- **Backlog:** Seleção inteligente por mood (Motor 5)
- Bundle `.ttf` files em `public/fonts/`

### Performance Strategy
- **Worker pool:** 4 concurrent renders (1 por variação)
- **Parallel execution:** Promise.all() para 4 variações
- **Target:** <8s total (vs 12s serial)
- **Memory management:** Explicit garbage collection + buffer cleanup

### Group 2: Product Image

```typescript
productImage: {
  url: string;        // Public URL or signed URL (Supabase Storage)
  targetBox?: {       // Bounding box of product (from Motor 1)
    x: number;        // normalized 0-1
    y: number;
    width: number;
    height: number;
  } | null;
}
```

**Transformations needed:**

1. **If `targetBox !== null`** (Motor 1 detected product):
   - Crop image to `targetBox` (normalized coordinates → pixels)
   - Resize to fit in `layout.productArea` without distortion (contain/cover)
   
2. **If `targetBox === null`** (product not detected):
   - Use entire image
   - Center in `layout.productArea`
   - Generate visual alert: "Produto não detectado — imagem completa usada"

**Libs:** Sharp for crop/resize (performance)

campaignData: {

  price: number | null;  // 8.99

**Formatting:**

## 🔒 Rendering Rules

### Rule 1: Layer Order (z-index)

```
1. Background (solid color #F5F5F5)
2. Product image (layout.productArea)
3. Decorative badges (decorative.priceBadge)
4. Text elements (layout.textArea, layout.priceArea)
5. Store identity (decorative.storeIdentity — logo or name)
6. Promotional title (decorative.promotionalTitle if exists)
```

### Rule 2: Text Overflow

If text exceeds `layout.textArea`:
- Apply word wrap (line break)
- If still exceeds height: truncate with "..." (ellipsis)
- Reduce fontSize by 10% and try again (max 2 iterations)

### Rule 3: Badge Shapes

Render 7 supported badge shapes:

| Shape | Description | SVG Path |
|-------|-----------|----------|
| `rounded-rect` | Rectangle with rounded corners (r=20px) | `roundRect()` |
| `cloud` | Irregular cloud (explosion) | Custom path |
| `star` | 5-pointed star | Custom path |
| `splash` | Splash/burst irregular | Custom path |
| `diamond` | Diamond | Rotate 45° rect |
| `oval` | Ellipse | `ellipse()` |
| `tag` | Tag with point | Custom path |

**Implementation:** Create `lib/ai/renderer/shapes.ts` with functions for each shape.

### Rule 4: Logo Positioning

If `decorative.storeIdentity.type === "logo"`:
- Load image from `logo_url`
- Resize to `decorative.storeIdentity.size` (maintain aspect ratio)
- Position at `decorative.storeIdentity.position`:
  - `top-left`: (40, 40)
  - `top-center`: ((1080 - width) / 2, 40)
  - `top-right`: (1080 - width - 40, 40)
  - `bottom-left`: (40, 1350 - height - 40)
  - `bottom-center`: ((1080 - width) / 2, 1350 - height - 40)
  - `bottom-right`: (1080 - width - 40, 1350 - height - 40)

If `logo_url === null`:
- Render `store_name` as text (font size 20px, weight 700)

## 📐 Critical Algorithms

### Algorithm 1: Crop & Resize Product Image

```typescript
async function cropAndResizeProduct(
  imageUrl: string,
  targetBox: {x, y, width, height} | null,
  productArea: {x, y, width, height}
): Promise<Buffer> {
  const img = await sharp(imageUrl);
  const metadata = await img.metadata();
  
  if (targetBox !== null) {
    // Crop to bounding box (normalized coordinates → pixels)
    const cropX = Math.round(targetBox.x * metadata.width!);
    const cropY = Math.round(targetBox.y * metadata.height!);
    const cropWidth = Math.round(targetBox.width * metadata.width!);
    const cropHeight = Math.round(targetBox.height * metadata.height!);
    
    img.extract({ left: cropX, top: cropY, width: cropWidth, height: cropHeight });
  }
  
  // Resize to fit in productArea (contain mode - no distortion)
  img.resize({
    width: productArea.width,
    height: productArea.height,
    fit: 'contain',
    background: { r: 0, g: 0, b: 0, alpha: 0 }, // transparent
  });
  
  return img.toBuffer();
}
```

### Algorithm 2: Render Badge Shape

```typescript
function renderBadge(
  ctx: CanvasRenderingContext2D,
  badge: {shape, position, size, rotation}
) {
  ctx.save();
  
  // Apply rotation if defined
  if (badge.rotation) {
    ctx.translate(badge.position.x + badge.size.width / 2, badge.position.y + badge.size.height / 2);
    ctx.rotate((badge.rotation * Math.PI) / 180);
    ctx.translate(-(badge.size.width / 2), -(badge.size.height / 2));
  } else {
    ctx.translate(badge.position.x, badge.position.y);
  }
  
  // Render specific shape
  switch (badge.shape) {
    case 'cloud':
      renderCloudShape(ctx, badge.size);
      break;
    case 'star':
      renderStarShape(ctx, badge.size);
      break;
    // ... other shapes
  }
  
  ctx.fillStyle = '#3B82F6'; // temporary neutral color
  ctx.fill();
  ctx.restore();
}
```

### Algorithm 3: Word Wrap with Ellipsis

```typescript
function renderTextWithWrap(
  ctx: CanvasRenderingContext2D,
  text: string,
  area: {x, y, width, height},
  fontSize: number
): void {
  ctx.font = `${fontSize}px Inter`;
  const words = text.split(' ');
  const lines: string[] = [];
  let currentLine = '';
  
  // Word wrap
  for (const word of words) {
    const testLine = currentLine + (currentLine ? ' ' : '') + word;
    const metrics = ctx.measureText(testLine);
    
    if (metrics.width > area.width) {
      lines.push(currentLine);
      currentLine = word;
    } else {
      currentLine = testLine;
    }
  }
  lines.push(currentLine);
  
  // Truncate if exceeds height
  const lineHeight = fontSize * 1.2;
  const maxLines = Math.floor(area.height / lineHeight);
  
  if (lines.length > maxLines) {
    lines.splice(maxLines);
    lines[maxLines - 1] += '...';
  }
  
  // Render lines
  lines.forEach((line, i) => {
    ctx.fillText(line, area.x, area.y + (i + 1) * lineHeight);
  });
}
```
