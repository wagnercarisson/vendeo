## 🎯 Objetivo

Implementar o **Motor 2 (Intent Resolver)** — o motor de decisão criativa que recebe dados estruturados e retorna a direção visual ideal para a composição da arte.

**O que ele faz:**
- Recebe `ImageProfile` (Motor 1) + `CampaignData` (banco) + `VisualSignature` (identidade da loja)
- Analisa as limitações da imagem, o objetivo da campanha e o DNA visual da loja
- Decide: hierarquia visual, mood, tratamento do produto, distribuição de texto, ênfase no preço, intensidade

**O que ele NÃO faz:**
- Não analisa imagens (isso é Motor 1)
- Não gera composições visuais (isso é Motor 3)
- Não renderiza artes (isso é Motor 4)

## 📥 Contrato de Entrada

### Grupo 1: `ImageProfile` (do Motor 1)

9 campos do output do Visual Reader:

| Campo | Tipo | Para que serve |
|-------|------|----------------|
| `backgroundType` | "transparent" \| "solid" \| "complex" \| "unknown" | Define directionTypes permitidos |
| `targetOccupancy` | "low" \| "medium" \| "high" \| "full" | `full` → proíbe background decorativo |
| `sceneType` | "single_product" \| "multiple_products" \| "lifestyle_scene" \| "full_scene" \| "unclear" | Hierarquia visual (único vs múltiplos) |
| `visibility` | "clear" \| "partial" \| "obstructed" \| "unknown" | Produto obstruído = evitar overlay |
| `framing` | "good" \| "tight" \| "distant" \| "unknown" | Distante = ampliar destaque |
| `imageQuality` | "good" \| "acceptable" \| "poor" \| "unknown" | Poor = composição simples |
| `matchType` | "exact" \| "category_only" \| "none" | `none` = estratégia diferente |
| `targetPosition` | "left" \| "center" \| "right" \| "top" \| "bottom" \| "mixed" \| "unknown" | Onde está produto → onde vai texto |
| `targetOrientation` | "vertical" \| "horizontal" \| "square" \| "mixed" \| "unknown" | Layout vertical vs horizontal |

**Campos do ImageProfile que NÃO são usados pelo Motor 2:**
- `targetBox` (usado pelo Motor 4 — Renderer)
- `subjectCutoff`, `visualIsolation`, `safeExpansionPotential`, `focusClarity` (relacionados a crop/recorte — fora do escopo)
- `reasoningSummary` (debug apenas)
- `detected`, `matchedTarget`, `confidence`, `relevantCount`, `ignoredElements` (contexto adicional, não regras)

### Grupo 2: `CampaignData` (do banco)

7 campos da tabela `campaigns`:

| Campo | Tipo | Para que serve |
|-------|------|----------------|
| `content_type` | "product" \| "service" \| "message" | Gate: "message" não processa (retorna direction vazia) |
| `objective` | "promocao" \| "novidade" \| "queima" \| "sazonal" \| "reposicao" \| "combo" \| "engajamento" \| "visitas" \| "informativo" \| "institucional" \| "autoridade" | Deriva context profile + influencia mood |
| `price` | number \| null | `!== null` → `priceEmphasis` aumenta |
| `price_label` | string \| null | Label customizado |
| `product_name` | string \| null | Contexto para LLM |
| `audience` | "geral" \| "jovens_festa" \| "familia" \| "premium_exigente" \| ... (12 valores) | Influencia mood |
| `product_positioning` | "popular" \| "medio" \| "premium" \| "jovem" \| "familia" | Influencia mood e intensity |

**Context Profile (derivado de `objective` via função):**

```typescript
function deriveContextProfile(objective: CampaignObjective): ContextProfile {
  switch (objective) {
    case "sazonal":
      return "seasonal";
    case "promocao":
    case "combo":
      return "promotional";
    case "queima":
      return "urgency";
    case "institucional":
    case "autoridade":
      return "premium";
    default: // novidade, reposicao, engajamento, visitas, informativo
      return "standard";
  }
}
```

| Context Profile | Quando | Características |
|----------------|--------|----------------|
| `standard` | Campanhas diárias comuns | Balanced, sem extremos |
| `promotional` | Promoções e combos | Strong intensity, high price emphasis |
| `seasonal` | Datas comemorativas | Playful mood, balanced |
| `premium` | Institucional, autoridade | Clean/premium mood, minimal intensity |
| `urgency` | Queima de estoque | Strong intensity, aggressive mood |

### Grupo 3: `VisualSignature` (do banco)

9 campos da tabela `visual_signatures` + `visual_signature_profiles`:

| Campo | Fonte | Tipo | Para que serve |
|-------|-------|------|----------------|
| `primary_color` | visual_signatures | text | Cor base da composição |
| `secondary_color` | visual_signatures | text | Cor de acento |
| `logo_url` | visual_signatures | text \| null | Logo da loja |
| `store_name_typography` | visual_signatures | jsonb `{font, weight}` | Fonte do nome da loja |
| `signature_seed` | visual_signatures | text (UUID) | Seed para consistência entre campanhas |
| `intensity_level` | visual_signature_profiles | "minimal" \| "balanced" \| "strong" | Base de `visualIntensity` |
| `composition_rules` | visual_signature_profiles | jsonb | Preferências de layout |
| `typography_rules` | visual_signature_profiles | jsonb | Regras de texto |
| `color_rules` | visual_signature_profiles | jsonb | Adaptação de cores |

**Busca do profile:**
1. Deriva `context_type` do `objective` (via função acima)
2. Busca `visual_signature` da loja (`store_id`)
3. Busca `visual_signature_profile` correspondente (`signature_id` + `context_type`)
4. Se não existir profile para esse contexto → usa "standard" como fallback

## 📤 Contrato de Saída

### `CreativeDirection` — 6 campos

```typescript
type CreativeDirection = {
  directionType: "hero" | "frame" | "split-dynamic" | "overlay" | "stacked";
  mood: "clean" | "aggressive" | "playful" | "premium";
  productTreatment: "framed" | "background";
  textDistribution: "left" | "right" | "center" | "overlay";
  priceEmphasis: "low" | "medium" | "high";
  visualIntensity: "minimal" | "balanced" | "strong";
};
```

| Campo | Valores | Como é decidido |
|-------|---------|----------------|
| `directionType` | hero / frame / split-dynamic / overlay / stacked | LLM decide + validação post-LLM via DIRECTION_TYPE_MAP |
| `mood` | clean / aggressive / playful / premium | LLM baseado em `audience` + `product_positioning` + context profile |
| `productTreatment` | framed / background | LLM decide + validação post-LLM via PRODUCT_TREATMENT_RULES |
| `textDistribution` | left / right / center / overlay | LLM baseado em `targetPosition` + `directionType` |
| `priceEmphasis` | low / medium / high | `price !== null` aumenta; context profile define magnitude |
| `visualIntensity` | minimal / balanced / strong | `intensity_level` do profile + ajustes por `imageQuality` |

---

### Descrição detalhada dos campos de saída

#### 1. `directionType` — Tipo de composição

| Valor | Quando usar | Exemplo visual |
|-------|------------|----------------|
| `hero` | Produto sozinho em destaque máximo | Produto centralizado, grande, resto é texto/preço |
| `frame` | Produto dentro de moldura/container | Produto emoldurado, texto ao redor |
| `split-dynamic` | Produto de um lado, texto do outro | Layout 50/50 ou 60/40 |
| `overlay` | Texto sobreposto à imagem | Produto ocupa tudo, texto por cima |
| `stacked` | Produto acima, texto abaixo (ou vice-versa) | Layout vertical empilhado |

#### 2. `mood` — Tom emocional

| Valor | Características |
|-------|----------------|
| `clean` | Minimalista, espaços brancos, elegante |
| `aggressive` | Cores fortes, contrastes altos, urgência |
| `playful` | Divertido, cores vibrantes, informal |
| `premium` | Sofisticado, tipografia refinada, sóbrio |

#### 3. `productTreatment` — Como tratar a imagem do produto

| Valor | Significado |
|-------|------------|
| `framed` | Produto fica dentro de área delimitada/moldura na composição |
| `background` | Imagem inteira é usada como fundo, texto/elementos por cima |

**Nota:** `"cutout"` (recorte do produto) foi removido — fora do escopo da Epic 4.

#### 4. `textDistribution` — Onde vai o texto

| Valor | Quando |
|-------|--------|
| `left` | Produto à direita, texto à esquerda |
| `right` | Produto à esquerda, texto à direita |
| `center` | Texto centralizado (produto acima/abaixo ou ao fundo) |
| `overlay` | Texto sobreposto à imagem |

#### 5. `priceEmphasis` — Quanto destacar o preço

| Valor | Quando |
|-------|--------|
| `low` | Sem preço, ou preço discreto |
| `medium` | Preço visível mas não dominante |
| `high` | Preço em destaque máximo (promoção) |

#### 6. `visualIntensity` — Intensidade geral da composição

| Valor | Características |
|-------|----------------|
| `minimal` | Poucas cores, muito espaço branco, clean |
| `balanced` | Equilíbrio entre elementos, standard |
| `strong` | Cores saturadas, muitos elementos, agressivo |

## 🔒 Regras de Validação Hard-Coded (Post-LLM)

O LLM gera uma `CreativeDirection` inicial, mas o sistema aplica regras determinísticas para garantir viabilidade.

### Tabela 1: DIRECTION_TYPE_MAP

Baseado no `backgroundType` da imagem, apenas certos `directionType` são permitidos:

| `backgroundType` | Allowed `directionType` | Lógica |
|-----------------|------------------------|--------|
| `transparent` | hero, split-dynamic, stacked, frame | PNG limpo — máxima liberdade |
| `solid` | frame, hero, split-dynamic | Fundo liso — composição direta |
| `complex` | overlay, frame, hero | Cena/ambiente — produto precisa de destaque |
| `unknown` | frame, hero | Fallback conservador |

**Regra de validação:**
```typescript
if (!DIRECTION_TYPE_MAP[profile.backgroundType].includes(direction.directionType)) {
  direction.directionType = DIRECTION_TYPE_MAP[profile.backgroundType][0]; // Usa primeiro da lista
}
```

---

### Tabela 2: PRODUCT_TREATMENT_RULES

Baseado no `backgroundType`, apenas certos tratamentos são viáveis:

| `backgroundType` | Allowed `productTreatment` | Lógica |
|-----------------|---------------------------|--------|
| `transparent` | framed, background | Ambos viáveis |
| `solid` | framed, background | Ambos viáveis |
| `complex` | background (apenas) | Cena completa — usar inteira |
| `unknown` | framed (apenas) | Fallback conservador |

**Regra de validação:**
```typescript
const rules = {
  transparent: ["framed", "background"],
  solid: ["framed", "background"],
  complex: ["background"],
  unknown: ["framed"],
};

if (!rules[profile.backgroundType].includes(direction.productTreatment)) {
  direction.productTreatment = rules[profile.backgroundType][0];
}
```

---

### Regra 3: targetOccupancy === "full"

Se o produto ocupa a imagem inteira, não há espaço para adicionar background decorativo atrás dele.

```typescript
if (profile.targetOccupancy === "full" && direction.directionType === "hero") {
  // Força uso de overlay ou frame — hero implica espaço ao redor
  direction.directionType = "overlay";
}
```

## 🤖 Modelo de IA

**Modelo:** `gpt-5.4` (full, não mini)

**Justificativa:**
- Motor 2 não processa imagem (só JSON estruturado)
- Raciocínio criativo requer modelo mais robusto
- Consistência nas decisões de `mood` e `directionType` é crítica
- Custo: ~$0.033/call vs ~$0.005 do mini (6.5x mais caro, mas vale a precisão)

**Configuração:**
```typescript
{
  model: "gpt-5.4",
  temperature: 0.3, // Criativo mas consistente
  timeout: 20000, // 20s
}
```

### Fluxo de execução

```typescript
// lib/ai/intent-resolver/service.ts
export async function resolveIntent(
  profile: ImageProfile,
  campaign: CampaignData,
  signature: VisualSignature
): Promise<CreativeDirection> {
  
  // 1. Gate: campanhas informativas não processam
  if (campaign.content_type === "message") {
    return emptyCreativeDirection();
  }

  // 2. Deriva context profile
  const contextType = deriveContextProfile(campaign.objective);
  
  // 3. Busca visual signature profile
  const visualProfile = await getVisualSignatureProfile(signature.id, contextType);

  // 4. Monta payload para o LLM
  const payload = {
    image: {
      backgroundType: profile.backgroundType,
      targetOccupancy: profile.targetOccupancy,
      sceneType: profile.sceneType,
      visibility: profile.visibility,
      framing: profile.framing,
      imageQuality: profile.imageQuality,
      matchType: profile.matchType,
      targetPosition: profile.targetPosition,
      targetOrientation: profile.targetOrientation,
    },
    campaign: {
      objective: campaign.objective,
      price: campaign.price,
      price_label: campaign.price_label,
      product_name: campaign.product_name,
      audience: campaign.audience,
      product_positioning: campaign.product_positioning,
    },
    identity: {
      primary_color: signature.primary_color,
      secondary_color: signature.secondary_color,
      signature_seed: signature.signature_seed,
      context_type: contextType,
      intensity_level: visualProfile.intensity_level,
      composition_rules: visualProfile.composition_rules,
      typography_rules: visualProfile.typography_rules,
      color_rules: visualProfile.color_rules,
    },
  };

  // 5. Chama LLM
  const raw = await callAI(
    [
      { role: "system", content: INTENT_RESOLVER_SYSTEM_PROMPT },
      { role: "user", content: JSON.stringify(payload, null, 2) },
    ],
    { model: "gpt-5.4", temperature: 0.3, timeoutMs: 20000 }
  );

  // 6. Valida e parseia
  const parsed = parseJsonFirstObject(raw);
  const validated = CreativeDirectionSchema.safeParse(parsed);
  
  if (!validated.success) {
    console.error("[intent-resolver] LLM output validation failed", validated.error);
    return buildFallbackDirection(profile, campaign);
  }

  // 7. Aplica regras hard-coded
  const final = applyValidationRules(profile, validated.data);

  return final;
}
```
