## User Story

**Como** equipe de engenharia do Vendeo,
**Quero** definir contratos Zod formalizados em `lib/domain/campaigns/contracts.ts` para validação de request/response dos endpoints de geração de campanhas,
**Para** garantir type-safety completo na boundary API, fechar o gap do endpoint de estratégia sem schema e habilitar testes unitários de contratos independentes de runtime.

---

## Objective

Criar `lib/domain/campaigns/contracts.ts` como fonte única de contratos de API para geração de campanhas, formalizando schemas Zod para request/response do endpoint principal e do endpoint de strategy, com testes unitários via `.safeParse()`.

---

## Scope IN

- Criar `lib/domain/campaigns/contracts.ts` com:
  - `GenerateCampaignRequestSchema` — re-export/alias de `CampaignRequestSchema` de `schemas.ts`
  - `GenerateCampaignResponseSchema` — Zod discriminated union espelhando `GenerateCampaignResult` de `service.ts`
  - `StrategyRequestSchema` — schema novo para `/api/generate/campaign/strategy` body (atualmente sem schema)
  - `StrategyResponseSchema` — schema novo para resposta de strategy (`suggestion: { audience, objective, productPositioning, reasoning }`)
  - Tipos TypeScript inferidos via `z.infer` para todos os schemas acima
- Criar `lib/domain/campaigns/contracts.test.ts` com testes unitários para cada schema (happy path + error cases)
- JSDoc com exemplos práticos de uso para cada schema exportado

## Scope OUT

- **Não** modificar `app/api/generate/campaign/route.ts` — continua usando `CampaignRequestSchema` de `schemas.ts`
- **Não** modificar `app/api/generate/campaign/strategy/route.ts` — adoção do novo schema é scope de Story 2.6
- **Não** criar contratos para `/api/generate/reels` ou `/api/generate/weekly-plan` — fora do escopo de campanhas
- **Não** criar `GenerateCampaignInputSchema` (input interno de `service.ts`) — é input de service layer, não API contract
- **Não** modificar `lib/campaigns/contracts.ts` (legacy layer) — arquivo diferente, já tem `@deprecated` de Story 2.2
- **Não** substituir imports existentes nos routes — migração para Story 2.6

## Acceptance Criteria

```gherkin
GIVEN que schemas.ts (Story 2.1) exporta CampaignRequestSchema com { campaign_id, force, persist, description }
WHEN contracts.ts exportar GenerateCampaignRequestSchema
THEN GenerateCampaignRequestSchema.safeParse({ campaign_id: "valid-uuid" }) retorna success: true
AND GenerateCampaignRequestSchema é re-export/alias de CampaignRequestSchema (sem duplicação de schema)
AND GenerateCampaignRequest type é inferido via z.infer
```

```gherkin
GIVEN que a API retorna { ok: true, requestId, reused, campaign_id?, output? } ou { ok: false, requestId, error, details?, status }
WHEN contracts.ts exportar GenerateCampaignResponseSchema como discriminated union
THEN .safeParse({ ok: true, requestId: "uuid", reused: false, campaign_id: "uuid" }) retorna success: true
AND .safeParse({ ok: false, requestId: "uuid", error: "CAMPAIGN_NOT_FOUND" }) retorna success: true
AND .safeParse({ ok: true }) retorna success: false (requestId ausente)
AND GenerateCampaignResponse type é inferido via z.infer
```

```gherkin
GIVEN que /api/generate/campaign/strategy recebe { product: { type, productName, description?, price? } }
WHEN contracts.ts exportar StrategyRequestSchema
THEN .safeParse({ product: { type: "product", productName: "Tênis Nike" } }) retorna success: true
AND .safeParse({ product: { type: "info", productName: "X" } }) retorna success: false (type inválido)
AND .safeParse({ product: { type: "service" } }) retorna success: false (productName ausente)
AND campo type usa z.enum(["product", "service", "message"]) — sem "info"
```

```gherkin
GIVEN que todos os schemas são definidos e exportados
WHEN npm run typecheck for executado
THEN saída é 0 erros
AND todos os tipos em contracts.ts são inferidos via z.infer (nenhum escrito manualmente)
AND import de Campaign, ContentType de types.ts funciona sem erros
```

```gherkin
GIVEN contracts.test.ts criado com vitest
WHEN npx vitest run lib/domain/campaigns/contracts.test.ts for executado
THEN todos os testes passam (happy path + error cases por schema)
AND coverage inclui: campo obrigatório ausente, enum inválido, type correto
```

## Dev Notes

### Estrutura real dos endpoints (descoberta via grep + read)

**`POST /api/generate/campaign`** (`app/api/generate/campaign/route.ts`):
- **Request:** Usa `CampaignRequestSchema` de `schemas.ts` — `{ campaign_id: uuid, force?: boolean, persist?: boolean, description?: string }`
- **Response OK:** `{ ok: true, requestId: string, reused: boolean, campaign_id?: string, output?: CampaignAIOutput }`
- **Response Error:** `{ ok: false, requestId: string, error: string, details?: unknown }`
- **Nota:** Request não contém dados do produto — campanha já existe no DB. A rota busca a campanha pelo `campaign_id`.

**`POST /api/generate/campaign/strategy`** (`app/api/generate/campaign/strategy/route.ts`):
- **Request:** `{ product: { type: string, productName: string, description?: string, price?: string } }` — **SEM schema Zod atualmente**
- **Response OK:** `{ ok: true, requestId: string, suggestion: { audience: string, objective: string, productPositioning: string, reasoning: string } }`
- **Response Error:** `{ ok: false, error: string }`

### Estrutura proposta para contracts.ts

```typescript
import { z } from "zod";
import { CampaignRequestSchema, CampaignAISchema } from "./schemas";

// ── Request ──────────────────────────────────────────────────
// Re-export do schema já existente em schemas.ts (sem duplicação)
export const GenerateCampaignRequestSchema = CampaignRequestSchema;
export type GenerateCampaignRequest = z.infer<typeof GenerateCampaignRequestSchema>;

// ── Response ─────────────────────────────────────────────────
const CampaignAIOutputSchema = CampaignAISchema.extend({
  price_label: z.string().nullable().optional(),
});

export const GenerateCampaignResponseSchema = z.discriminatedUnion("ok", [
  z.object({
    ok: z.literal(true),
    requestId: z.string(),
    reused: z.boolean(),
    campaign_id: z.string().optional(),
    output: CampaignAIOutputSchema.optional(),
  }),
  z.object({
    ok: z.literal(false),
    requestId: z.string(),
    error: z.string(),
    details: z.unknown().optional(),
  }),
]);
export type GenerateCampaignResponse = z.infer<typeof GenerateCampaignResponseSchema>;

// ── Strategy Request ─────────────────────────────────────────
export const StrategyRequestSchema = z.object({
  product: z.object({
    type: z.enum(["product", "service", "message"]),
    productName: z.string().min(1),
    description: z.string().optional(),
    price: z.string().optional(),
  }),
});
export type StrategyRequest = z.infer<typeof StrategyRequestSchema>;

// ── Strategy Response ─────────────────────────────────────────
export const StrategySuggestionSchema = z.object({
  audience: z.string(),
  objective: z.string(),
  productPositioning: z.string(),
  reasoning: z.string(),
});

export const StrategyResponseSchema = z.discriminatedUnion("ok", [
  z.object({
    ok: z.literal(true),
    requestId: z.string(),
    suggestion: StrategySuggestionSchema,
  }),
  z.object({
    ok: z.literal(false),
    error: z.string(),
  }),
]);
export type StrategyResponse = z.infer<typeof StrategyResponseSchema>;
```
