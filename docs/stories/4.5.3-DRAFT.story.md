---

## 🎯 Problem

### Current Behavior (After Story 4.5.2)

```typescript
// Visual Composer (Motor 3) OUTPUT:
{
  typography: {
    productName: {
      fontSize: 48,
      fontWeight: "700",
      fontFamily: "Montserrat",  // ← GPT chose this
      color: "#FF0000",           // ← GPT chose this
      lineHeight: 1.2              // ← GPT chose this
    }
  }
}

// Motor 4 (Renderer) IGNORES new fields:
ctx.font = `700 48px "Inter"`;  // ← Hardcoded font
ctx.fillStyle = "#1F2937";       // ← Hardcoded color
const lineHeight = 48 * 1.2;     // ← Hardcoded calculation
```

**Result:** Compositions look generic despite GPT's intelligent font/color choices.

### Evidence

**Investigation Finding (Story 4.5.2):**
- Motor 4 code search: NO references to `fontFamily`, `color`, `lineHeight` from spec
- Constants file: `RENDERER_FONT_FAMILY = "Inter"` (hardcoded)
- Canvas rendering: Uses `RENDERER_COLORS.text` (hardcoded)

**User Impact:**
- 🟡 MEDIUM — Functional but aesthetically suboptimal
- All variations use same font/color (less variety)
- GPT's creative typography choices are wasted

---

## 🏗️ Solution Approach (DRAFT)

### Option A: Dynamic Typography Rendering (RECOMMENDED)

**Strategy:**
1. Update `lib/ai/renderer/canvas.ts` to read fontFamily/color/lineHeight from spec
2. Fallback to hardcoded constants if fields missing (backward compatible)
3. Add font loading logic (if custom fonts needed)
4. Update tests to validate new rendering

**Pros:**
- Leverages GPT's intelligent typography choices
- Backward compatible (optional fields)
- Improves composition variety

**Cons:**
- Font loading complexity (may need font registry)
- Performance impact if fonts not cached
- Color validation needed (ensure contrast/accessibility)

**Effort:** 4-5 hours

---

### Option B: Font/Color Registry (More Complex)

**Strategy:**
1. Create font registry with approved fonts (Montserrat, Sora, Inter, etc.)
2. Create color palette with brand-safe colors
3. Motor 4 validates GPT choices against registry
4. Fallback to closest match if not in registry

**Pros:**
- Brand consistency guaranteed
- Performance optimized (pre-loaded fonts)
- Accessibility validation built-in

**Cons:**
- More complex implementation
- Limits GPT creativity
- Registry maintenance overhead

**Effort:** 6-8 hours
