## Objective

**Problema:** Endpoint `/campaign/strategy` existe desde antes do Epic 2 e não foi atualizado para usar a arquitetura de contratos criada nas Stories 2.1-2.3:
- ❌ `const { product } = await req.json()` — aceita qualquer JSON (sem validação)
- ❌ `JSON.parse(content)` sem schema — IA pode retornar `objective: "invalid_value"` e o sistema aceita
- ❌ `StrategySuggestionSchema` usa `z.string()` frouxo — não aproveita os enums reais de `schemas.ts`
- ❌ Schemas de validação (`CampaignAudienceSchema`, etc.) não são exportados — impossível importar

**Solução:** 3 mudanças cirúrgicas:
1. Exportar 3 schemas internos de `schemas.ts`
2. Fortalecer `StrategySuggestionSchema` em `contracts.ts` com enums estritos + exportar como `StrategyAIOutputSchema`
3. Ativar validação no endpoint com `safeParse` (400 request inválido, 500 IA inválida)

## Dev Notes

### Pattern 1: Exportar schemas internos de schemas.ts

```typescript
// lib/domain/campaigns/schemas.ts
// Adicionar 'export' nas 3 constantes internas (sem mudar lógica):

export const CampaignObjectiveSchema = z.preprocess(
  (value) => {
    if (value == null || value === "") return null;
    const normalized = normalizeObjective(String(value));
    return normalized || value;
  },
  z.enum(OBJECTIVE_VALUES).nullable()
);

// AUDIENCE_VALUES e POSITIONING_VALUES também precisam ser exportados:
export const AUDIENCE_VALUES = AUDIENCE_OPTIONS.map((option) => option.value) as [
  (typeof AUDIENCE_OPTIONS)[number]["value"],
  ...(typeof AUDIENCE_OPTIONS)[number]["value"][]
];

export const POSITIONING_VALUES = PRODUCT_POSITIONING_OPTIONS.map(
  (option) => option.value
) as [
  (typeof PRODUCT_POSITIONING_OPTIONS)[number]["value"],
  ...(typeof PRODUCT_POSITIONING_OPTIONS)[number]["value"][]
];

export const CampaignAudienceSchema = z.preprocess(
  (value) => {
    if (value == null || value === "") return null;
    return value;
  },
  z.enum(AUDIENCE_VALUES).nullable()
);

export const ProductPositioningSchema = z.preprocess(
  (value) => {
    if (value == null || value === "") return null;
    return value;
  },
  z.enum(POSITIONING_VALUES).nullable()
);
```

**Atenção:** Exportar `AUDIENCE_VALUES` e `POSITIONING_VALUES` é necessário para o `z.enum()` em `StrategyAIOutputSchema`. Sem isso, `contracts.ts` não pode construir o enum de forma type-safe.

### Pattern 2: StrategyAIOutputSchema (enums estritos SEM z.preprocess)

```typescript
// lib/domain/campaigns/contracts.ts — ATUALIZAR StrategySuggestionSchema
// IMPORTANTE: NÃO usar CampaignObjectiveSchema aqui (tem z.preprocess para normalização de input legacy)
// IA retorna valores canônicos diretamente — validar com enum simples:

import { AUDIENCE_VALUES, OBJECTIVE_VALUES, POSITIONING_VALUES } from "./schemas";

/**
 * Valida a resposta bruta da IA para sugestão de estratégia.
 * Usa enums estritos (sem preprocess) pois IA deve retornar valores canônicos.
 * Diferente de CampaignAudienceSchema/CampaignObjectiveSchema que têm preprocess para legacy.
 */
export const StrategyAIOutputSchema = z.object({
  audience: z.enum(AUDIENCE_VALUES),
  objective: z.enum(OBJECTIVE_VALUES),
  productPositioning: z.enum(POSITIONING_VALUES),
  reasoning: z.string().min(10, "Reasoning deve ter no mínimo 10 caracteres"),
});
export type StrategyAIOutput = z.infer<typeof StrategyAIOutputSchema>;
```

### Pattern 3: Ativar validação no endpoint (request + AI response)

```typescript
// app/api/generate/campaign/strategy/route.ts
import { StrategyRequestSchema, StrategyAIOutputSchema } from "@/lib/domain/campaigns/contracts";

export async function POST(req: Request) {
  const openai = getOpenAI();
  const requestId = crypto.randomUUID();

  try {
    // 1. Validar request (NEW)
    const json = await req.json().catch(() => null);
    const body = StrategyRequestSchema.safeParse(json);

    if (body.success === false) {
      return NextResponse.json(
        { ok: false, requestId, error: "INVALID_INPUT", details: body.error.flatten() },
        { status: 400 }
      );
    }

    const { storeId } = await getUserStoreIdOrThrow();
    const { product } = body.data; // tipado e validado

    const prompt = `...${product.productName}...`; // usar body.data em vez de req.json() direto

    const ai = await openai.chat.completions.create({ ... });

    // 2. Validar resposta da IA (NEW)
    const content = ai.choices[0].message.content || "{}";
    const parsed = JSON.parse(content);
    const validated = StrategyAIOutputSchema.safeParse(parsed);

    if (validated.success === false) {
      console.error("[strategy] AI returned invalid output:", validated.error.flatten());
      return NextResponse.json(
        { ok: false, requestId, error: "AI_INVALID_OUTPUT", details: validated.error.flatten() },
        { status: 500 }
      );
    }

    return NextResponse.json({
      ok: true,
      requestId,
      suggestion: validated.data, // tipado: StrategyAIOutput
    });

  } catch (err: any) {
    const msg = typeof err?.message === "string" ? err.message : "UNKNOWN_ERROR";
    console.error("[strategy] error:", msg, err?.stack ?? err);
    return NextResponse.json(
      { ok: false, requestId, error: "UNHANDLED", details: msg },  // requestId aqui também
      { status: 500 }
    );
  }
}
```

### Atenção: z.preprocess vs z.enum direto

| Schema | Onde usar | Por quê |
|--------|-----------|---------|
| `CampaignObjectiveSchema` (preprocess) | `DbCampaignSchema`, `CampaignDomainSchema` | Normaliza input legacy do usuário (ex: "lancamento" → "novidade") |
| `z.enum(OBJECTIVE_VALUES)` direto | `StrategyAIOutputSchema` | IA retorna valores canônicos — não precisa normalizar |

Se usar `CampaignObjectiveSchema` (com preprocess) no `StrategyAIOutputSchema`, valores inválidos são normalizados em vez de rejeitados — comportamento incorreto para validação da IA.

### Nota: Frontend compatibility

O frontend atual em `CampaignCard.tsx`, `CampaignEditForm.tsx`, etc. usam `import * as selectors from "@/lib/domain/campaigns/selectors"`. O endpoint `/strategy` é chamado pelo `onboarding/` ou pelo novo form de campanha. **@dev deve verificar** qual componente faz o POST para `/api/generate/campaign/strategy` e confirmar que o payload já inclui `{ product: { type, productName } }` conforme `StrategyRequestSchema`.
