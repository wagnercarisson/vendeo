## 📋 Overview

This document defines 22 fallback templates for Motor 2 (Intent Resolver) badges and CTAs across Phase 1 contexts. 

**Purpose:** Guarantee brand-appropriate badge text when AI generation fails or returns invalid content.

**Scope:**
- 5 contexts (mutually exclusive)
- 22 templates total
- Phase 1 only (promotional, urgency, premium, standard, seasonal generic)
- Phase 2+ (seasonal-specific: Natal, Black Friday, etc.)

Every template is validated against:

```
Badge constraints:
├─ MAX 30 characters (including spaces)
├─ Portuguese (BR) natural language

└─ Contextually appropriate tone

CTA constraints:
├─ MAX 25 characters (including spaces)
├─ Tone-aligned (urgent/elegant/neutral)
├─ Action-oriented (verb-first preferred)
└─ Brand-neutral (no Vendeo-specific language)
```

## 🔧 Implementation Notes

### Character Validation
All 22 templates have been validated:
- ✅ Badges: MAX 30 characters
- ✅ CTAs: MAX 25 characters

**Tool for validation:**
```typescript
// Badge validation
function validateBadge(text: string): boolean {
  return text.length <= 30 && text.trim().length > 0;
}

// CTA validation
function validateCTA(text: string): boolean {
  return text.length <= 25 && text.trim().length > 0;
}
```

---

### Integration Pattern

```typescript
// In Motor 2 (Intent Resolver)
import FALLBACK_TEMPLATES from '@/lib/fallback-templates';

function getFallbackBadge(visual_context: VisualContext): string | null {
  const templates = FALLBACK_TEMPLATES[visual_context]?.badges;
  if (!templates || templates.length === 0) return null;
  
  // Randomly select from context-specific badges
  return templates[Math.floor(Math.random() * templates.length)];
}

function getFallbackCTA(visual_context: VisualContext, tone: CTATone): string | null {
  const templates = FALLBACK_TEMPLATES[visual_context]?.ctas;
  if (!templates || templates.length === 0) return null;
  
  // Tone is derived from visual_context at campaign creation
  return templates[Math.floor(Math.random() * templates.length)];
}
```

### Fallback Logic (Motor 2 Error Handling)

```
Campaign creation flow:
1. User selects visual_context (dropdown)
2. Motor 1: ImageProfile extracted
3. Motor 2: Intent Resolver generates badges
   └─ If AI response is:
      ├─ Valid & contextual → Use AI response
      ├─ Invalid (>30 chars, wrong tone) → Use FALLBACK_TEMPLATES[context]
      └─ NULL/error → Use FALLBACK_TEMPLATES[context]
4. Motor 3: Visual Composer renders
5. Motor 4: Output to 4 variations
```

**Success metric:** Fallback usage <20% (if exceeds, Motor 2 needs review)
