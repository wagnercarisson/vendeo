## 📋 REQUIREMENTS

### **1. Badge Positioning Matrix**

Define positioning rules for 3 badge positions:

| Position | Context | Visual Rationale | Canvas Coordinates |
|----------|---------|------------------|-------------------|
| **top-left** | promotional | High visibility, attention-grabbing | x: 20, y: 20 |
| **top-center** | urgency, standard | Centered focus, neutral | x: canvas.width/2, y: 20 |
| **top-right** | premium | Subtle elegance, non-intrusive | x: canvas.width - 150, y: 20 |

**Requirements:**
- Specify exact canvas coordinates for each position
- Account for badge width (variable based on text length)
- Define margins/padding (from edges, from product image)
- Provide responsive rules (different canvas sizes: 1080×1080, 1080×1920)

### **2. Visual Hierarchy Rules**

Define rendering priority and overlaps:

**Hierarchy (z-index):**
1. Badge (foreground)
2. CTA button (mid-ground)
3. Product image (background)
4. Background color/gradient (base)

**Overlap rules:**
- Badge MUST NOT overlap product image key areas (face, text, logo)
- CTA button MUST NOT overlap badge
- Badge shadow/glow for legibility on light backgrounds

### **3. Color Selection Rules**

Define badge background colors per context:

| Context | Badge Color | Rationale | Contrast Ratio |
|---------|-------------|-----------|----------------|
| promotional | `#FF6B6B` (vibrant red) | Urgency, attention | ≥4.5:1 (WCAG AA) |
| urgency | `#FFA500` (orange) | Action, time-sensitive | ≥4.5:1 |
| premium | `#2C3E50` (dark navy) | Elegance, exclusivity | ≥7:1 (WCAG AAA) |
| standard | `#3498DB` (blue) | Neutral, trustworthy | ≥4.5:1 |

**Additional rules:**
- Text color: white for all badges (legibility)
- Badge opacity: 95% (slight transparency for elegance)
- Border radius: 8px (rounded corners, not pill-shaped)

### **4. Typography Rules**

Define text rendering for badges:

**Font selection:**
- Primary: `Inter Bold` (if available)
- Fallback: `Arial Bold` (system font)

**Font size:**
- Badge text: 18px (for 1080×1080), 24px (for 1080×1920)
- CTA text: 16px (for 1080×1080), 20px (for 1080×1920)

**Text constraints:**
- Max badge text: 30 characters (enforced by Motor 2)
- Max CTA text: 25 characters (enforced by Motor 2)
- Text alignment: center (within badge)
- Text padding: 12px horizontal, 8px vertical

### **5. Example Compositions (3-5 Mockups)**

Provide visual examples showing:

**Example 1: Promotional Badge + CTA**
- Product: Chocolate Nestlé 150g
- Badge: "25% OFF Páscoa" (top-left, red)
- CTA: "Compre 3, Leve 4" (bottom-center)
- Canvas: 1080×1080
- Background: gradient (light to dark)
- Product image: centered, 60% canvas height

**Example 2: Urgency Badge Only**
- Product: iPhone 15 Pro
- Badge: "Últimas 5 unidades" (top-center, orange)
- No CTA
- Canvas: 1080×1920
- Background: solid dark blue
- Product image: top 40%, text bottom 60%

**Example 3: Premium CTA Only**
- Product: Vinho Reserva
- No badge
- CTA: "Exclusivo para você" (bottom-right)
- Canvas: 1080×1080
- Background: dark gradient
- Product image: 70% canvas, left-aligned

**Mockup format:**
- Annotated image (PNG or Figma link)
- Canvas coordinates labeled
- Color hex codes shown
- Font sizes noted

## ⚠️ CONSTRAINTS

**Phase 1 Scope (MUST RESPECT):**
- ✅ Promotional badges + CTAs
- ✅ Urgency badges + CTAs
- ✅ Premium CTAs only (no badges)
- ✅ Standard CTAs only (no badges)
- ❌ Seasonal badges + CTAs = **Phase 2** (define but mark as Phase 2)

**Technical Constraints:**
- Badge width: dynamic (based on text length), max ~200px
- Canvas sizes: 1080×1080 (square), 1080×1920 (story)
- Rendering tech: HTML5 Canvas API (via fabric.js or similar)
- Text rendering: Canvas `fillText()` method

**Visual Constraints:**
- WCAG AA contrast (≥4.5:1) for all badges
- No overlap with product key areas (face, logo, text)
- Badge must not occlude more than 15% of product image
