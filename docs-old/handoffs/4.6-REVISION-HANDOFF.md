## 📋 CONTEXT — Why Revision is Needed

### Original Story 4.6 Problem
The original approach used **hardcoded badge texts** ("MEGA OFERTA", "COMPRE AGORA") that:
- ❌ Always repeat the same jargon
- ❌ Are generic and not product-specific
- ❌ Don't adapt to campaign context/instructions
- ❌ Require constant code maintenance for new badges

### New Approach (User-Validated Decision)
Use **Motor 2 (Intent Resolver)** to generate contextual badges/CTAs:
- ✅ AI reads campaign data + user instructions (description field)
- ✅ Generates product-specific texts: "Última Coca 2L: R$ 5,90" (not "MEGA OFERTA")
- ✅ Adapts CTA to audience: "Reserve sua garrafa" (premium) vs "Garanta já" (promotional)
- ✅ Infinite creativity without hardcoded consts

## 🧠 TECHNICAL DECISIONS (Already Made)

### Decision 1: Badge Text Generation
**Problem:** How to generate discount badges ("50% OFF") without discount data?  
**Solution:** Motor 2 generates contextual generic badges based on `visual_context` + `description` field

| Context | Badge Example |
|---------|---------------|
| promotional | "Última Coca 2L: R$ 5,90" |
| urgency | "Só hoje: Hambúrguer especial" |
| seasonal | "Especial de Natal" |
| premium | (none - minimalism) |
| standard | (none) |

### Decision 2: CTA Badge Presence
**Problem:** Should CTA be present in premium contexts?  
**Solution:** Always present, but TEXT VARIES by profile to match audience sophistication

| Profile | CTA Example | Tone |
|---------|-------------|------|
| promotional | "Garanta já" | Urgent, aggressive |
| premium | "Reserve sua garrafa" | Elegant, inviting |
| urgency | "Corra" | Scarcity |
| standard | "Garanta o seu" | Neutral |

### Decision 3: Seasonal Theme Detection
**Problem:** Where does seasonal theme come from?  
**Solution:** Auto-detect by date + UI checkbox for opt-out

**Schema addition needed:**
```sql
ALTER TABLE campaigns ADD COLUMN use_seasonal_theme boolean DEFAULT NULL;
-- NULL = auto-detect, TRUE = force, FALSE = disable
```

**UI:**
- Checkbox appears when seasonal theme detected (ex: Dec 15-31 = Natal)
- Default: enabled (user can uncheck to disable)
- Tooltip: "Selecione para habilitar temas diferenciados para datas comemorativas"

### Decision 4: Description Field Purpose
**Problem:** Field has no clear purpose, users don't know what to write  
**Solution:** Rename to "Detalhes / Instruções" + add tooltip explaining it guides AI

**UI Change:**
```
Label: "Detalhes / Instruções"
Placeholder: "Descreva características, contexto ou instruções para a campanha"
Tooltip: "Use este campo para guiar a IA. Ex: 'Última unidade disponível', 
         'Produto artesanal premium', 'Promoção de fim de semana'"
```

### Prompt Engineering Requirement

**Why @prompt-eng is Critical:**
Motor 2 prompt must be carefully engineered to:
1. Generate SHORT texts (max 30 chars) to avoid truncation
2. Include product name when relevant (avoid generic jargon)
3. Match audience tone (elegant for premium, urgent for promotional)
4. Parse user instructions from description field
5. Respect visual context (promotional vs premium vs urgency)

**Prompt Sections to Add:**
```
<badge_generation>
Rules for generating promotional badge:
- Short (max 25 chars)
- Include product name OR key detail
- Urgency words: "Última", "Só hoje", "Últimas unidades"
- Avoid generic: "MEGA OFERTA", "PROMOÇÃO"
- Examples: "Última Coca 2L: R$ 5,90", "Só hoje: Hambúrguer especial"

Rules for generating CTA badge:
- Action verb + benefit (max 25 chars)
- Match audience tone:
  * Premium → "Reserve", "Conheça", "Explore"
  * Promotional → "Garanta", "Corra", "Aproveite"
  * Standard → "Garanta o seu", "Peça agora"
- Examples: "Reserve sua garrafa", "Garanta já"
</badge_generation>

<campaign_instructions>
Parse "description" field for intent keywords:
- Urgency: "última unidade", "estoque limitado", "só hoje" → use urgent tone
- Premium: "exclusivo", "edição limitada", "artesanal" → use elegant tone
- Promotional: "promoção", "oferta", "desconto" → use aggressive tone
</campaign_instructions>
```

### What Art Director Must Provide

**Deliverable:** `docs/stories/4.6-VISUAL-COMPOSITION-GUIDE.md`

**Contents:**

1. **Badge Positioning Matrix**
   ```
   | Badge Type | Position | Rationale |
   |------------|----------|-----------|
   | Promotional | top-left | Urgency = first thing user sees |
   | Seasonal | top-center | Thematic = centered symmetry |
   | CTA | bottom-center | Action zone = call to action |
   ```

2. **Visual Hierarchy Rules**
   - Product > Text > Badge (never badge > product)
   - Max 3 badges on screen (more = cluttered)
   - Badge size: 160-200x50-60px (promotional), 120-140x40-50px (seasonal)

3. **Color Selection Rules**
   - Promotional badge: `secondary_color` (store accent) + high saturation
   - Urgency badge: `#DC2626` (red alert) — override store colors
   - Seasonal badge: Theme colors (ex: Natal = `#c41e3a` red)
   - CTA badge: Context-dependent (urgent = red, inviting = green, elegant = gold)

4. **Typography Rules**
   - Promotional: Bold (700) or Black (900)
   - Seasonal: Semibold (600) or Bold (700)
   - CTA urgent: Black (900)
   - CTA elegant: Semibold (600)

5. **Example Compositions (Visual Mockups)**
   - Coca Cola 2L promotional (badge top-left + CTA bottom-center)
   - Wine premium (no badge + elegant CTA)
   - Seasonal Natal (seasonal badge top-center + inviting CTA)
